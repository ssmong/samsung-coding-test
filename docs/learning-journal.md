## 현재 강점 / 약점 요약

**강점**
- 문제 지문의 동사 순서를 그대로 코드 순서로 옮기는 습관 (3190에서 "머리→벽→몸→사과/꼬리" 순 정확 이행)

## 자주 놓치는 패턴

> 2회 이상 반복된 실수만 등재.
> **해소 판정:** 단순히 N문제 지나간 게 아니라, 그 패턴이 **적용 가능했던 문제**에서 3회 연속 안 나와야 해소.
> "적용 가능 = 그 버그를 노출시킬 수 있는 유형의 문제" (예: BFS 패턴은 BFS 문제에서만 카운트).

| 패턴 | 적용 가능 카테고리 | 발생 | 마지막 발생 | 적용 후 무발생 | 상태 |
|---|---|---:|---|---:|---|
| dr/dc(행/열) 변수 오타 | 격자 이동 전체 | 2 | 2026-04-16 (14503) | 0/3 | 관찰중 |
| BFS visited를 popleft 시점에 표시 | bfs, multi-source bfs | 0 | - | 0/3 | - |
| `input()` 사용 (sys.stdin.readline 누락) | 전체 | 0 | - | 0/3 | - |
| 시뮬레이션 동시 변경 시 스냅샷 누락 | simulation | 0 | - | 0/3 | - |

> "적용 후 무발생" 컬럼: `현재 무발생 카운트 / 해소 임계값(3)`. 3/3에 도달하면 상태 → 해소, 다음 압축 시 제거 가능.
