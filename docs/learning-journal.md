## 현재 강점 / 약점 요약

업데이트 일자: 2026-05-06

**강점**
- 문제 지문의 동사 순서를 그대로 코드 순서로 옮기는 습관 (3190에서 "머리→벽→몸→사과/꼬리" 순 정확 이행)
- board 값을 상수로 추상화 (`EMPTY/APPLE/SNAKE`, `RIPE/UNRIPE/WALL`) → 분기가 바로 읽힘
- BFS 첫 문제(7576)에서 dist=-1 sentinel 통합 + 멀티소스 시작점 일괄 enqueue 정확히 적용

**약점**
- 행/열 혼동 — 변수 오타(`hr,hr`, `c += dr[d]`)에서 경계 체크 뒤집힘(`nx<m, ny<n`)으로 형태 바뀌며 3회 연속 발생. **격자 이동 코드 작성 직후 "행 인덱스 < 행 개수, 열 인덱스 < 열 개수" 한 번 더 점검 루틴 필요**
- BFS 후 답 추출 단계 검증 부족 — 시작점(dist=0)을 "안 익음"으로 오해, inner break만 사용, early return 누락, `max(2D list)` 오용이 한 답 계산에 동시 발생
- 분기 간 공통 코드를 미리 추출하지 않는 경향 (APPLE/else 중복)
- 상수 정의 후 코드에서 매직 넘버로 직접 비교 — 일관성 부족 (7576: `RIPE` 정의했지만 `grid[..]==0` 매직넘버)

## 자주 놓치는 패턴

> 2회 이상 반복된 실수만 등재.
> **해소 판정:** 단순히 N문제 지나간 게 아니라, 그 패턴이 **적용 가능했던 문제**에서 3회 연속 안 나와야 해소.
> "적용 가능 = 그 버그를 노출시킬 수 있는 유형의 문제" (예: BFS 패턴은 BFS 문제에서만 카운트).

| 패턴 | 적용 가능 카테고리 | 발생 | 마지막 발생 | 적용 후 무발생 | 상태 |
|---|---|---:|---|---:|---|
| 행/열 혼동 (변수 오타 또는 경계 체크 뒤집힘) | 격자 이동 전체 | 3 | 2026-05-06 (7576) | 0/3 | 관찰중 |
| 위치 인덱스 매핑 오류 (시계방향 0~7 vs 직관적 12/3/6/9) | 회전 구조 (톱니, 주사위, 원판) | 1 | 2026-05-05 (14891) | 1/3 | 관찰중 |
| 공백 없는 숫자 문자열을 `.split()`으로 파싱 시도 | 격자/문자열 입력 | 1 | 2026-05-05 (14891) | 1/3 | 관찰중 |
| BFS visited를 popleft 시점에 표시 | bfs, multi-source bfs | 0 | - | 1/3 | - |
| `input()` 사용 (sys.stdin.readline 누락) | 전체 | 0 | - | 1/3 | - |
| 시뮬레이션 동시 변경 시 스냅샷 누락 | simulation | 0 | - | 0/3 | - |

> "적용 후 무발생" 컬럼: `현재 무발생 카운트 / 해소 임계값(3)`. 3/3에 도달하면 상태 → 해소, 다음 압축 시 제거 가능.
