# 삼성 SW 역량테스트 (코딩테스트) 정리

## 시험 개요

삼성의 SW 역량테스트는 소프트웨어 직군 지원자를 대상으로 하는 실기 형태의 코딩 테스트로 GSTA를 대체한다. 온라인이 아닌 지정된 고사장 PC에서 직접 프로그램을 작성하는 방식이다.

- **시험 시간:** 4시간 (240분)
- **문제 수:** 총 2개 문제 (각 문제당 10개 테스트케이스)
- **사용 언어:** C, C++, Java, Python
- **시험 환경:** 지정 고사장에서 오프라인 시행, 오리엔테이션 진행 후 백지(연습장) 제공. Python은 PyCharm, Java는 Eclipse 제공.
- **준비물:** 수험표, 신분증(운전면허/주민등록증/여권), 볼펜
- **인터넷 및 촬영 불가:** 외부 자료 열람, 사진 촬영 등 제한. 외부 라이브러리 사용 불가(Python 표준 라이브러리만 가능).

## 등급 체계 (A / B / C)

삼성 SW 역량테스트는 A형(Advanced), B형(Professional), C형(Expert) 3등급으로 구분된다.

**A형** — 공채에서 치르는 시험. 복잡한 조건을 정확하게 구현하는 능력을 평가한다. 1문제 맞추면 A등급, 2문제 다 맞추면 A+ 등급. 시뮬레이션, BFS/DFS, 완전탐색 유형이 주를 이루며, Python 사용 가능.

**B형** — A형보다 문제 이해 자체는 간단하지만, 제약 조건(입력 크기 등)이 매우 커서 효율적인 알고리즘 설계와 자료구조 활용이 필수. 세그먼트 트리, KMP 등 고급 알고리즘이 출제되기도 한다. 구현해야 하는 부분과 테스트 부분(main)이 분리되어 제공되는 형태. SSAFY 또는 삼성 알고리즘 특강 참여자에 한해 응시 기회가 주어진다.

**C형** — 삼성 내부 SW 개발자 중 B형 소유자만 응시 가능. C/C++만 사용 가능. 사실상 일반 개발자가 도달 가능한 최고 등급은 B형.

→ **공채 지원자는 A형에 집중하면 된다.**

## 난이도

- 백준 solved.ac 기준, 최근 삼성 기출의 최저 난이도는 골드4, 최고 난이도는 플래티넘3까지 나온 적 있다.
- 1번 문제(합격선): 골드3~골드1 수준을 시간 내에 안정적으로 풀 수 있으면 통과 가능.
- 2번 문제: 알고리즘 + 최적화가 필요한 고난도. 플래티넘 하위 수준.
- 대부분의 기업 코딩테스트가 실버5~골드2 범위에서 출제된다.

## 종합면접 구성

삼성 공채 면접은 3가지로 구성된다:

- **임원 면접:** 개인 품성, 조직 적합성 평가
- **직무역량 면접:** 전공 역량, 직무 동기 평가
- **창의성 면접:** 독창적 아이디어와 논리 전개 과정 평가

최종 면접 합격자에 한해 건강검진을 실시하며, 건강검진 합격자만 최종 합격 및 입사가 가능하다.

## 언어 선택 관련

- Python은 인터프리터 언어라 C++보다 실행 속도가 느리며, 시간 제한이 빡빡한 문제에서 시간 초과(TLE) 위험이 있다.
- 삼성 B형부터는 과거 Python 사용이 불가능했으나, 최근 변경되어 사용 가능. 다만 A형에서도 Python 시간 보정이 넉넉하지 않다는 후기가 많다.
- Python 사용 시 `sys.stdin.readline`, `sys.setrecursionlimit` 등 최적화가 필수.
- 미래로봇추진단 시스템소프트웨어 직무는 C/C++가 실무 언어이므로, 여유가 되면 C++ 병행 학습 권장.

## 핵심 출제 유형 상세 분석

삼성 SW 역량테스트는 단순한 알고리즘 지식 검증이 아닌, **복잡한 조건을 빠르고 정확하게 구현하는 실전 능력**을 평가한다. A형 기준 출제 알고리즘은 크게 브루트 포스, DFS, BFS 세 가지가 핵심이며, 이들이 시뮬레이션과 결합되어 나온다.

### 1. 시뮬레이션 + 구현

삼성 코테의 가장 대표적인 유형. 문제에서 주어진 규칙과 조건을 그대로 코드로 옮기는 것이 핵심이다. 특정 알고리즘 지식보다는 **구현력과 꼼꼼함**이 요구된다.

**자주 등장하는 구현 패턴:**
- **2차원 격자(NxN 배열) 위에서의 이동:** 상하좌우 4방향 또는 대각선 포함 8방향 이동. 방향 벡터(`dx`, `dy` 배열)를 사용하여 처리.
- **2D 배열 회전:** 시계/반시계 90도 회전, 특정 영역만 회전하는 변형.
- **나선형(토네이도) 탐색:** 2차원 배열에서 나선 형태로 순회.
- **복잡한 상태 전이:** 매 턴마다 여러 객체가 동시에 이동/충돌/생성/소멸하는 시뮬레이션. 순서가 중요하며, 한 턴의 처리 과정을 단계별로 정확히 나눠야 한다.
- **다수의 조건 분기:** 문제 설명이 길고 조건이 많아서, 하나라도 빠뜨리면 오답이 되는 구조.

**대표 기출 예시(백준 기준):** 주사위 굴리기, 뱀, 미세먼지 안녕!, 낚시왕, 마법사 상어 시리즈

**유도제어/시뮬레이터 경험자에게 유리한 점:** 상태를 정의하고 매 스텝마다 갱신하는 구조가 제어 시뮬레이션과 유사. 다만 코테에서는 "조건 해석의 정확성"이 핵심이므로 문제를 꼼꼼히 읽는 습관이 중요.

### 2. BFS (너비 우선 탐색)

삼성 기출에서 **가장 꾸준히 등장**하는 알고리즘. 격자 위에서 최단 거리/최소 횟수를 구하거나, 영역을 확산시키는 문제에 사용된다.

**삼성 코테에서 BFS가 나오는 전형적인 형태:**
- **격자 위 최단 경로:** NxN 맵에서 출발점→도착점까지 벽을 피해 이동하는 최소 이동 횟수.
- **동시 확산:** 불이 번지면서 동시에 사람도 이동해야 하는 문제. BFS를 두 개 운용하거나 시간 순서를 고려해야 한다.
- **시뮬레이션과 결합:** 매 턴마다 BFS로 탐색하고 그 결과를 바탕으로 다음 상태를 결정하는 복합 문제. 최근 기출에서는 이 형태가 많다.
- **다차원 상태 BFS:** 단순한 (x, y)가 아니라 (x, y, 벽 부순 횟수) 또는 (x, y, 열쇠 보유 상태) 등 추가 상태를 포함하여 탐색.

**대표 기출 예시:** 미로 탐색, 벽 부수고 이동하기, 아기 상어, 메두사와 전사들

### 3. DFS + 백트래킹 (완전 탐색)

가능한 모든 경우를 탐색하되, 가지치기(pruning)로 불필요한 탐색을 줄이는 방식. 삼성 코테에서는 **조합/순열을 생성**하거나 **특정 조건을 만족하는 배치를 찾는** 문제로 나온다.

**삼성 코테에서 DFS/백트래킹이 나오는 전형적인 형태:**
- **N개 중 M개 선택:** 벽을 세울 위치 3곳 선택, 감시카메라 방향 조합 등.
- **순열/조합 생성 후 시뮬레이션:** 가능한 모든 조합을 만들고, 각 조합에 대해 시뮬레이션을 돌려서 최적값을 구하는 구조. 전형적인 "브루트 포스 + 시뮬레이션" 패턴.
- **맵 탐색:** 격자에서 연결된 영역 찾기, 섬의 개수 세기 등.

**대표 기출 예시:** 연구소(벽 3개 세우기 + BFS), 감시(카메라 방향 조합), 사다리 조작, 테트로미노

### 4. 최근 트렌드 (2020년 이후)

과거에는 시뮬레이션/구현만으로 풀 수 있는 문제가 많았지만, 최근에는 **알고리즘과 시뮬레이션의 복합** 문제가 증가하고 있다.

- 시뮬레이션 + BFS 결합 문제가 주류
- 2번 문제에서 Tree DP, Segment Tree, Priority Queue 등 고급 자료구조 활용 문제 출제
- 구현량 자체도 늘어나 코드 길이가 길어지는 추세
- 제출 시간이 평가 요소에 반영된다는 후기 있음

## 준비 전략

### 핵심 개념·템플릿 정리

- 삼성 기출에서 자주 등장하는 주요 알고리즘(DFS·BFS·완전탐색)과 자료구조(스택·큐·그래프)별 코드 템플릿 작성
- 자주 쓰는 입출력·시뮬레이션 코드 템플릿 숙지: 방향 벡터, 2D 배열 회전, 격자 범위 체크 등
- 코드트리 또는 백준의 삼성 코딩 테스트 기출문제를 풀고 해설과 함께 코드 비교

### 실전 감각 끌어올리기

- 시간 제한을 두고 모의코딩테스트 실시
- 틀린 문제 집중 복습: 패턴·실수 원인 분석
- '시뮬레이션' 유형 추가 풀이, 입력 처리 로직 완성도 높이기

### 안정성 체크

- 실전과 동일한 시간 제한으로 모의고사 진행
- 제출 전 주요 디버깅 체크리스트 작성 및 적용: 경계값, 예외 케이스, 자료형 범위 확인
- 언어별 최적화 훈련:
  - C++ → fast I/O, reserve 메모리 사용
  - Python → pypy, sys.stdin.readline 등 입력 속도 개선

### 실전 팁

- 기초 자료구조·알고리즘 로직을 직접 손으로 구현해 볼 것
- 입력 처리, 디버깅, 예외 처리 등 실무형 습관을 기르는 것이 합격에 직결
- 시험장 긴장으로 인해 평소보다 낮은 퍼포먼스를 보이는 경우가 많으므로, 타이머를 켠 모의 테스트로 실전 감각을 몸에 익히는 것이 중요
- 문제풀이 후 코드 리뷰나 오답 분석으로 사고 과정을 점검할 것
