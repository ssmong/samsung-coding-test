# deque로 회전(rotate) 구현

## 언제 쓰나
- 원형 배열을 시계/반시계 한 칸씩 회전시킬 때
- 양끝 삽입·삭제가 잦을 때 (`list.pop(0)` 회피)

## 코드
```python
from collections import deque

g = deque([1, 0, 1, 0, 1, 1, 1, 1])  # 12시부터 시계방향 인덱스

g.rotate(1)   # 시계방향 1칸 (오른쪽으로 밀림): 마지막 원소가 0번으로
g.rotate(-1)  # 반시계 1칸 (왼쪽으로 밀림): 0번 원소가 마지막으로

# 부호 = 방향과 그대로 매핑되므로, 회전 방향 변수를 그대로 넘기면 됨
direction = -1  # -1: 반시계, +1: 시계
g.rotate(direction)
```

## 함정
- **list 슬라이싱(`g = [g[-1]] + g[:-1]`)도 가능하지만 매번 새 리스트 생성** → 회전 자주 하면 deque가 빠르고 명확.
- **인덱스 헷갈림** — 12시=`[0]`, 3시=`[2]`, 6시=`[4]`, 9시=`[6]`. 옆 톱니/원판과 비교할 때 인덱스 한 칸만 틀려도 전체 오답.
- **동시 회전 문제** — 여러 객체가 서로의 상태를 보고 회전 여부를 결정할 때, "한 객체 돌리고 즉시 옆에 전파"하면 안 됨. 회전 방향 배열에 모두 결정한 뒤 마지막에 일괄 `rotate()` 호출.
- `rotate()`는 in-place 수정. 반환값 없음 (`None`).
