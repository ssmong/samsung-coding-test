# 주사위 굴리기 (6면 회전)

## 언제 쓰나
주사위나 정육면체를 격자 위에서 동/서/남/북으로 굴릴 때. 면 번호(1~6)를 추적하는 게 아니라 **방향 슬롯을 고정하고 값만 순환**시키는 게 핵심.

## 핵심 아이디어
주사위 상태를 "어느 면이 위냐"로 저장하지 말고, **"각 방향 슬롯에 어떤 값이 있냐"**로 저장.

인덱스 고정: `0=top, 1=bottom, 2=north, 3=south, 4=east, 5=west`

굴릴 때는 4개 슬롯만 순환 (진행 축과 수직인 2개는 불변).

## 코드
```python
# 인덱스: 0=top, 1=bot, 2=N, 3=S, 4=E, 5=W
dice = [0] * 6

def roll(dice, d):
    t, b, n, s, e, w = dice
    if d == 0:    # 동: top→E→bot→W→top (W가 top으로)
        dice[0], dice[4], dice[1], dice[5] = w, t, e, b
    elif d == 2:  # 서
        dice[0], dice[5], dice[1], dice[4] = e, t, w, b
    elif d == 3:  # 북: top→N→bot→S→top (S가 top으로)
        dice[0], dice[2], dice[1], dice[3] = s, t, n, b
    elif d == 1:  # 남
        dice[0], dice[3], dice[1], dice[2] = n, t, s, b
```

## 함정
- **회전 방향 헷갈림**: 동쪽으로 굴리면 **왼쪽(서)면이 위로** 올라온다. "굴러가는 방향의 반대면이 top이 된다"로 외우자.
- **면 번호 ≠ 값**: "모든 면에 0이 쓰여 있다"는 뜻은 `dice = [0]*6`. 1~6은 "면 이름"일 뿐 초기값 아님.
- **명령 번호와 방향 인덱스 불일치**: BOJ 14499는 1=동,2=서,3=북,4=남. 표준 동남서북(0~3)과 다르니 매핑 테이블(`[-1,0,2,3,1]`) 써서 더미 0번으로 밀면 깔끔.
- **경계 밖이면 굴리기도 생략**: `continue`로 `roll()` 호출 자체를 건너뛴다. 출력도 안 함.
