# 격자 문제 고정 세팅

## 언제 쓰나

NxN/NxM 격자 위에서 탐색·시뮬레이션하는 모든 문제. 세 가지(인덱싱·경계 체크·방향 벡터)를 매번 같은 꼴로 박아두면 시험장에서 생각할 자원을 문제 로직에 집중할 수 있다.

**스타일 메모:**
- `DR`, `DC`는 변하지 않는 상수라서 모듈 레벨 + 대문자. 함수 어디서든 읽어도 안전.
- `n`, `board` 같은 데이터는 인자로 전달, 결과는 return.
- 모든 로직을 `main()`으로 감싸면 진짜 전역은 import와 상수뿐.

### 방향 번호 ↔ 이동량 표 (회전 버전)

| d | 방향 | dr[d] | dc[d] |
|---|---|---|---|
| 0 | 우 | 0 | +1 |
| 1 | 하 | +1 | 0 |
| 2 | 좌 | 0 | -1 |
| 3 | 상 | -1 | 0 |

```
    0:우
  3:상   1:하     ← 시계방향 순환
    2:좌
```

시계방향으로 번호가 올라가므로 **"오른쪽 회전 = +1"** 이 성립. `(3 + 1) % 4 == 0`이 상→우로 자연스럽게 순환시킨다.

## 함정

- **1-indexed vs 0-indexed는 문제 지문에 맞춰라.** 지문이 "r행 c열" 1-based인데 0-indexed로 받으면 매 입력마다 `-1` 보정이 쌓여 off-by-one 지옥. 3190·16236 등은 1-based.
- **거리·방문 배열은 0-indexed로 두고 입력만 변환**하는 것도 가능하지만, 본 코드 안에서 인덱싱이 두 종류 섞이면 혼란. 한 문제에선 하나로 통일.
- **방향 벡터 배치 순서는 목적에 따라 다르다.** BFS는 아무 순서 OK, **회전 문제는 반드시 시계방향 순**으로 배치해야 `±1 % 4` 공식이 성립. 상어·뱀·청소기 계열 필수.
- **`in_range` 함수 호출 오버헤드**는 N ≤ 100, 스텝 ≤ 10^5이면 무시 가능. 10^7 넘어가면 인라인 고려.
- **Python 음수 모듈로는 양수로 돌아옴** (`-1 % 4 == 3`). 회전 공식에 `(d - 1) % 4`를 바로 써도 안전.
